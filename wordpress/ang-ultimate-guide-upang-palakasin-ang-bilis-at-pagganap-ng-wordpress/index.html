Bakit ang Bilis ay Mahalaga para sa Iyong WordPress Site?

Ipinakikita ng mga pag-aaral na mula 2000 hanggang 2016, ang karaniwang laki ng pansin ng tao ay bumaba mula sa 12 segundo hanggang 7 segundo.

Ano ang ibig sabihin nito para sa iyo bilang isang may-ari ng website?

Mayroon kang napakaliit na oras upang ipakita ang mga gumagamit ng iyong nilalaman at kumbinsihin ang mga ito upang manatili sa iyong website.

Ang isang mabagal na website ay nangangahulugang ang mga gumagamit ay maaaring umalis sa iyong website bago ito kahit na naglo-load.

Ayon sa isang pag-aaral ng kaso sa StrangeLoop na kasama ang Amazon, Google, at iba pang mas malaking site, ang isang ikalawang pagkaantala sa oras ng pag-load ng pahina ay maaaring humantong sa 7% pagkawala sa mga conversion, 11% na mas kaunting mga pagtingin sa pahina, at 16% na pagbawas sa kasiyahan ng customer.

Higit pa rito, ang Google at iba pang mga search engine ay nakapagsimula na na parusahan ang mas mabagal na mga website sa pamamagitan ng pagtulak sa kanila sa mga resulta ng paghahanap na nangangahulugan ng mas mababang trapiko para sa mabagal na mga website.

Upang buuin ang lahat ng ito, kung nais mo ng mas maraming trapiko, tagasuskribi, at kita mula sa iyong website, dapat mong gawin ang iyong WordPress website FAST!

Paano Suriin ang Bilis ng iyong WordPress Website?

Kadalasan ang mga nagsisimula ay nag-iisip na ang kanilang website ay OK lamang dahil hindi ito nararamdaman na mabagal sa kanilang computer. Iyan ay isang malaking pagkakamali.

Dahil madalas mong bisitahin ang iyong sariling website, ang mga modernong browser tulad ng Chrome ay nag-iimbak ng iyong website sa cache at awtomatikong i-prefetch ito sa lalong madaling simulan mo ang pag-type ng isang address. Ginagawang halos agad-load ang iyong website.

Sa katunayan, ang mga gumagamit sa iba’t ibang mga heograpikal na lokasyon ay magkakaroon ng isang ganap na iba’t ibang karanasan.

Ito ang dahilan kung bakit inirerekumenda namin na subukan mo ang bilis ng iyong website gamit ang isang tool tulad ng Pingdom.

Ito ay isang libreng online na tool na nagbibigay-daan sa iyo upang subukan ang bilis ng iyong website mula sa iba’t ibang mga lokasyon.

Pagkatapos mong patakbuhin ang bilis ng pagsubok ng iyong website, maaari kang magtanong kung ano ang isang mahusay na bilis ng website na dapat kong layunin para sa?

Ang isang mahusay na oras ng pagkarga ng pahina ay nasa ilalim ng 2 segundo.

Gayunpaman, ang mas mabilis na maaari mong gawin ito, mas mabuti ito. Ang ilang mga milliseconds ng pagpapabuti dito at doon ay maaaring magdagdag ng hanggang sa pag-ahit off ang kalahati o kahit isang buong ikalawang mula sa iyong oras ng pag-load.

Ano ang Pinapabagal ng Iyong Website sa WordPress?

Ang iyong ulat sa pagsubok ng bilis ay malamang na magkaroon ng maramihang mga rekomendasyon para sa pagpapabuti. Gayunpaman karamihan sa mga iyon ay teknikal na pananalita na mahirap para sa mga nagsisimula na maunawaan.

Gayunpaman ang pag-unawa kung ano ang slows down ang iyong website ay susi sa pagpapabuti ng pagganap at paggawa ng mas matalinong pang-matagalang desisyon.

Ang mga pangunahing dahilan para sa isang mabagal na website ng WordPress ay:

  • Web Hosting – Kapag ang iyong web hosting server ay hindi maayos na naka-configure maaari itong saktan ang bilis ng iyong website.
  • Pagsasaayos ng WordPress – Kung ang iyong WordPress site ay hindi naghahain ng mga naka-cache na pahina, ito ay sobrang karga ng iyong server kaya nagiging sanhi ng iyong website na maging mabagal o ganap na pag-crash.
  • Laki ng pahina – Pangunahing mga larawan na hindi na-optimize para sa web.
  • Mga Bad Plugin – Kung gumagamit ka ng isang mahina naka-code na plugin, maaari itong makabuluhang pabagalin ang iyong website.
  • Panlabas na mga script – Ang mga panlabas na script tulad ng mga ad, loader ng font, atbp ay maaari ring magkaroon ng malaking epekto sa pagganap ng iyong website.

Ngayon na alam mo kung ano ang slows down ang iyong WordPress website, tingnan natin kung paano mapabilis ang iyong website sa WordPress.

Mag-install ng WordPress Caching Plugin

Ang mga pahina ng WordPress ay “dynamic.” Ito ay nangangahulugang ang mga ito ay binuo sa mabilisang tuwing may isang taong bumibisita sa isang post o pahina sa iyong website. Upang bumuo ng iyong mga pahina, ang WordPress ay kailangang magpatakbo ng isang proseso upang mahanap ang kinakailangang impormasyon, ilagay ang lahat ng sama-sama, at pagkatapos ay ipakita ito sa iyong user.

Ang prosesong ito ay nagsasangkot ng maraming mga hakbang, at maaari itong makapagpabagal ng iyong website kapag maraming tao ang dumadalaw sa iyong site nang sabay-sabay.

Iyon ang dahilan kung bakit inirerekumenda namin ang bawat WordPress site na gumamit ng caching plugin. Ang Caching ay maaaring gawing kahit saan ang iyong WordPress site mula 2x hanggang 5x.

Narito kung paano ito gumagana: Sa halip na dumaan sa buong proseso ng pagbuo ng pahina sa bawat oras, ang iyong caching plugin ay gumagawa ng isang kopya ng pahina pagkatapos ng unang pagkarga, at pagkatapos ay naglilingkod na naka-cache na bersyon sa bawat kasunod na gumagamit.

Tulad ng makikita mo sa mga graphics sa itaas, kapag bumisita ang isang user sa iyong WordPress site, na binuo gamit ang PHP, ang iyong server ay nakakakuha ng impormasyon mula sa isang database ng MySQL at ang iyong mga file na PHP, at pagkatapos ay ang lahat ay magkasama sa isang HTML na nilalaman na hinahain sa user. Ito ay isang mahabang proseso, ngunit maaari mong laktawan ang isang pulutong ng mga ito kapag gumamit ka ng caching sa halip.

Mayroong maraming mga caching plugin na magagamit para sa WordPress, ngunit inirerekumenda namin ang paggamit ng WP Super Cache plugin. Tingnan ang aming gabay sa hakbang sa hakbang kung paano i-install at i-setup ang WP Super Cache sa iyong WordPress site. Hindi mahirap i-set up, at mapapansin ng iyong mga bisita ang pagkakaiba.

Tandaan: Kung gumagamit ka ng pinamamahalaang hosting provider ng hosting, hindi mo na kailangan ang caching plugin dahil inaalagaan nila ito para sa iyo.

I-optimize ang Mga Imahe para sa Bilis

Ang mga imahe ay nagdadala ng buhay sa iyong nilalaman at makakatulong na mapalakas ang pakikipag-ugnayan. Natuklasan ng mga mananaliksik na ang paggamit ng mga may kulay na visual ay gumagawa ng 80% ng mga tao na mas malamang na basahin ang iyong nilalaman.

Ngunit kung ang iyong mga imahe ay hindi na-optimize, maaari silang masakit kaysa sa pagtulong. Sa katunayan, ang di-na-optimize na mga imahe ay isa sa mga pinakakaraniwang isyu ng bilis na nakikita natin sa mga website ng mga nagsisimula.

Bago ka mag-upload ng isang larawan nang direkta mula sa iyong telepono o camera, inirerekumenda namin na gumamit ka ng software sa pag-edit ng larawan upang i-optimize ang iyong mga larawan para sa web.

Sa kanilang orihinal na mga format, ang mga larawang ito ay maaaring magkaroon ng malaking laki ng file. Ngunit batay sa format ng file ng imahe at ang compression pipiliin mo sa iyong software sa pag-edit, maaari mong bawasan ang laki ng iyong imahe sa pamamagitan ng hanggang sa 5x.

Gumagamit lamang kami ng dalawang format ng imahe: JPEG at PNG.

Ngayon ay maaaring ikaw ay nagtataka: ano ang kaibahan?

Buweno, PNG Hindi na-compress ang format ng imahe. Kapag nag-compress ka ng isang imahe ito ay nawawala ang ilang impormasyon, kaya ang isang hindi na-compress na imahe ay magiging mas mataas na kalidad na may higit pang detalye. Ang downside ay na ito ay isang mas malaking laki ng file, kaya ito ay tumatagal ng mas mahaba upang i-load.

JPEG , sa kabilang banda, ay isang naka-compress na format ng file na bahagyang binabawasan ang kalidad ng imahe, ngunit mas maliit ito sa laki.

Kaya paano tayo magpasya kung anong format ng imahe ang pipiliin?

  • Kung ang aming larawan o imahe ay may maraming iba’t ibang kulay, pagkatapos ay ginagamit namin ang JPEG.
  • Kung ito ay isang mas simpleng larawan o kailangan namin ng isang transparent na imahe, gagamitin namin ang PNG.

Ang karamihan ng aming mga imahe ay JPEG.

Gaya ng nakikita mo sa chart, ang format ng imahe na iyong ginagamit ay maaaring gumawa ng malaking pagkakaiba sa pagganap ng iyong website.

Gumamit ng Content Delivery Network (CDN)

Tandaan kung paano namin nabanggit sa itaas na ang mga gumagamit sa iba’t ibang mga heyograpikal na lokasyon ay maaaring makaranas ng iba’t ibang beses sa paglo-load sa iyong site?

Iyan ay dahil ang lokasyon ng iyong mga web hosting server ay maaaring magkaroon ng epekto sa bilis ng iyong site. Halimbawa, sabihin nating ang iyong web hosting company ay may mga server nito sa Estados Unidos. Ang isang bisita na nasa Estados Unidos ay karaniwang makikita ang mas mabilis na oras ng paglo-load kaysa sa isang bisita sa India.

Ang paggamit ng isang CDN, o Content Delivery Network, ay makakatulong upang mapabilis ang oras ng paglo-load para sa lahat ng iyong mga bisita.

Ang CDN ay isang network na binubuo ng mga server sa buong mundo. Ang bawat server ay mag-iimbak ng “static” na mga file na ginamit upang makabuo ng iyong website. Ang mga static na file ay walang pagbabago na mga file tulad ng mga imahe, CSS, at JavaScript, hindi tulad ng iyong mga pahina ng WordPress na “dynamic” gaya ng ipinaliwanag sa itaas.

Kapag gumamit ka ng isang CDN, sa tuwing dumalaw ang isang user sa iyong website, sila ay nagsilbi sa mga static na file mula sa alinmang server ay pinakamalapit sa kanila. Ang iyong sariling web hosting server ay magiging mas mabilis dahil ang CDN ay gumagawa ng maraming trabaho.

Huwag Mag-upload ng Mga Video nang direkta sa WordPress

Ikaw maaari direktang mag-upload ng mga video sa iyong WordPress site, at awtomatiko itong maipakita ang mga ito sa isang HTML5 player …

Ngunit hindi mo dapat gawin iyon!

Ang pag-host ng mga video ay magdudulot sa iyo ng bandwidth. Maaari kang singilin ng mga bayarin sa overage sa pamamagitan ng iyong web hosting company, o maaari pa nilang i-shut down ang iyong site nang sama-sama, kahit na ang iyong plano ay may kasamang “walang limitasyong” bandwidth.

Ang pag-host ng mga video ay pinatataas din ang iyong mga sukat ng backup nang napakahusay, at ginagawang mahirap para sa iyo na ibalik ang WordPress mula sa backup.

Sa halip, dapat kang gumamit ng serbisyo sa pag-host ng video tulad ng YouTube, Vimeo, DailyMotion, atbp, at hayaan silang pangalagaan ang hirap. Mayroon silang bandwidth para dito!

Ang WordPress ay may built-in na video na naka-embed na tampok, kaya maaari mong kopyahin at ilagay mismo ang URL ng iyong video sa iyong post at awtomatiko itong i-embed.

Alamin ang higit pang mga detalye kung paano ito gumagana sa aming gabay sa pag-embed ng mga video sa WordPress.

Bawasan ang mga panlabas na HTTP Requests

Maraming mga plugin ng WordPress at mga tema ang nag-load ng lahat ng mga uri ng mga file mula sa iba pang mga website. Ang mga file na ito ay maaaring magsama ng mga script, mga estilo ng estilo, at mga imahe mula sa mga panlabas na mapagkukunan tulad ng Google, Facebook, mga serbisyo ng analytics, at iba pa.

Ito ay ok na gumamit ng ilan sa mga ito. Marami sa mga file na ito ang na-optimize upang i-load nang mabilis hangga’t maaari, kaya mas mabilis kaysa sa pagho-host ng mga ito sa iyong sariling website.

Ngunit kung ang iyong mga plugin ay gumawa ng maraming mga kahilingan na ito, maaaring mabagal ang iyong website nang malaki.

Maaari mong bawasan ang lahat ng mga panlabas na kahilingan ng HTTP sa pamamagitan ng hindi pagpapagana ng mga script at estilo o pagsasama sa mga ito sa isang file. Narito ang isang tutorial kung paano i-disable ang mga file at JavaScript ng iyong plugin.

Sa kasamaang palad, maraming mga hindi maganda ang naka-code na tema ng WordPress out doon. Binabalewala nila ang mga pamantayang pamantayan ng WordPress at nagtatapos ang paggawa ng direktang mga tawag sa database, o masyadong maraming hindi kinakailangang mga kahilingan sa database. Ito ay maaaring makapagpabagal ng iyong server sa pamamagitan ng pagbibigay ng masyadong maraming trabaho upang gawin.

Kahit na mahusay na naka-code na mga tema ay maaaring magkaroon ng code na gumagawa ng mga tawag sa database upang makuha ang pangunahing impormasyon ng iyong blog.

Sa halimbawang ito, sa bawat oras na nakikita mo
charset = ">

Hindi mo maaaring sisihin ang mga developer ng tema para sa na. Wala silang ibang paraan upang malaman kung anong wika ang iyong site.

Ngunit kung pinapasadya mo ang iyong site gamit ang tema ng bata, maaari mong palitan ang mga tawag na ito ng database gamit ang iyong partikular na impormasyon upang mabawasan ang lahat ng mga tawag sa database. 





Suriin ang iyong tema ng magulang para sa mga pagkakataon tulad nito na maaaring madaling mapalitan ng static na impormasyon. 




Optimize WordPress Database 








Pagkatapos ng paggamit ng WordPress sa isang sandali, ang iyong database ay magkakaroon ng maraming impormasyon na malamang na hindi mo na kailangan.  Para sa pinabuting pagganap, maaari mong i-optimize ang iyong database upang mapupuksa ang lahat ng hindi kinakailangang impormasyon. 




Madali itong mapadali gamit ang WP-Sweep plugin.  Pinapayagan ka nitong linisin ang iyong database ng WordPress sa pamamagitan ng pagtanggal ng mga bagay tulad ng mga na-trashed na post, mga pagbabago, mga hindi nagamit na mga tag, atbp. I-optimize din ang istraktura ng iyong database sa isang pag-click lamang. 






Limitasyon ang Mga Post Revision 








Mag-post ng mga pagbabago sa mga puwang sa iyong database ng WordPress.  Ang ilang mga gumagamit ay naniniwala na ang mga pagbabago ay maaaring makaapekto sa ilang mga query sa database na pinapatakbo ng mga plugin.  Kung hindi partikular na ibukod ng plugin ang mga pagbabago sa post, maaari itong pabagalin ang iyong site sa pamamagitan ng paghahanap sa mga ito nang hindi kinakailangan. 




Madali mong limitahan ang bilang ng mga pagbabago na pinanatili ng WordPress para sa bawat artikulo.  Idagdag lamang ang linyang ito ng code sa iyong wp-config.php file. 




tukuyin ('WP_POST_REVISIONS', 4); 




Ang batas na ito ay maglilimita sa WordPress upang i-save lamang ang iyong huling 4 na pagbabago ng bawat post o pahina, at awtomatikong itatapon ang mas lumang mga pagbabago. 




Huwag paganahin ang Hotlinking at Leaching ng Iyong Nilalaman 








Kung gumagawa ka ng nilalamang may kalidad sa iyong WordPress na site, ang malungkot na katotohanan ay marahil ito ay makakuha ng ninakaw sa lalong madaling panahon. 




Ang isang paraan na nangyari ito ay kapag ang iba pang mga website ay nagsilbi nang direkta sa iyong mga larawan mula sa kanilang mga URL sa iyong website, sa halip na i-upload ang mga ito sa kanilang sariling mga server.  Sa katunayan, ang mga ito ay pagnanakaw ng iyong web hosting bandwidth, at hindi ka makakakuha ng anumang trapiko upang ipakita ito. 




Idagdag lamang ang code na ito sa iyong .htaccess file upang harangan ang hotlinking ng mga larawan mula sa iyong WordPress site. 




#disable hotlinking ng mga imahe na may ipinagbabawal o custom na pagpipilian ng imahe
 RewriteEngine on
 RewriteCond% {HTTP_REFERER}! ^ $
 RewriteCond% {HTTP_REFERER}! ^ Http (s)?: // (www.)? Site.com [NC]
 RewriteCond% {HTTP_REFERER}! ^ Http (s)?: // (www.)? Google.com [NC]
 RewriteRule. (Jpg | jpeg | png | gif) $ - [NC, F, L] 




Tandaan: Huwag kalimutang palitan ang site.com gamit ang iyong sariling domain. 




Maaari mo ring suriin ang aming artikulo na nagpapakita ng 4 mga paraan upang maiwasan ang pagnanakaw ng imahe sa WordPress. 




Ang ilang mga nilalaman ng pag-scrap ng mga website awtomatikong lumikha ng mga post sa pamamagitan ng pagnanakaw ng iyong nilalaman mula sa iyong RSS feed.  Maaari mong suriin ang aming gabay sa pagpigil sa pag-scrap ng nilalaman ng blog sa WordPress para sa mga paraan upang makitungo sa awtomatikong pagnanakaw ng nilalaman.
